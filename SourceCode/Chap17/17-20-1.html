Window 객체

Window 객체의 주요 속성 및 메서드
속성 설명
document Document 객체를 반환한다.
history History 객체를 반환한다.
location Location 객체를 반환한다.
navigator Navigator 객체를 반환한다.
screen Screen 객체를 반환한다.
opener 현재의 Window를 열어준(open해준) 부모 Window 객체를 참조한다.
메서드 설명
alert() 알림창을 띄운다.
blur() Window가 포커스를 잃도록 설정한다.
clearInterval() setInterval() 메서드에 의해 시작된 코드의 반복 실행을 멈춘다.
clearTimeout() setTimeout() 메서드에 의해 연기된 코드의 실행을 취소한다.
close() Window를 닫는다.
confirm() 확인창을 띄운다. 확인창은 메시지, <확인> 버튼 및 <취소> 버튼으로 이루어져 있다.
focus() Window가 포커스를 갖도록 설정한다.
moveBy() Window를 현재의 위치에서 지정된 위치만큼 이동시킨다(상대적 이동).
moveTo() Window를 지정된 위치로 이동시킨다(절대적 이동).
open() 새로운 Window를 연 후에 새로 열린 Window 객체를 반환한다.
사용법은 window.open(URL, name, specs, replace)와 같은데 각 매개변수에 대한 자세한 설명은 아래의 표를 참고하자.
참고로 open() 메서드에서 사용되는 모든 매개변수(URL, name, specs, replace)는 생략이 가능하다.
매개변수 셜명
URL 열려고 하는 페이지의 URL을 지정한다. 만약 URL을 지정하지 않으면 about:blank 페이지가 열린다.
name target 특성을 지정하거나 Window의 이름을 지정한다. 지정 가능한 값들을 정리하면 아래와 같다.
  • _blank : 새로운 Window를 열고 URL을 로드한다.[기본값]
  • _parent : 부모 프레임에 URL을 로드한다.
  • _self : 현재 페이지에 URL을 로드한다.
  • _top : URL을 로드할 수 있는 프레임셋에 URL을 로드한다.
  • name : 임의로 지정하는 Window의 이름이다. 만약 여기서 지정한 이름(name)과 동일한 이름을 가진 Window가 이미 열려 있으면
    열려 있는 해당 Window에 URL을 로드하고, 없으면 여기서 지정한 이름(name)을 가지는 새로운 Window를 열고 URL을 로드한다.
specs 열려질 Window의 사양(specs)을 지정한다. 사양에서 지정 가능한 항목들은 아래와 같다.
항목 지정 가능한 값 설명
channelmode yes, no, 1, 0. 참고로 yes는 1과 같고 no는 0과 같다. 극장 모드로 보여줄 것인지의 여부를 지정한다. IE에서만 사용가능. [기본값:no]
directories yes, no, 1, 0 디렉터리 버튼의 추가 여부를 지정한다. IE에서만 사용가능. [기본값:yes]
fullscreen yes, no, 1, 0 전체 화면으로 보여줄 것인지의 여부를 지정한다. IE에서만 사용가능. [기본값:no]
height 픽셀 Window의 높이를 지정한다. 최소값은 100이다.
left 픽셀 Window의 좌측 위치(x 좌표)를 지정한다.
location yes, no, 1, 0 주소창의 보임 여부를 지정한다.
menubar yes, no, 1, 0 메뉴바의 보임 여부를 지정한다.
resizable yes, no, 1, 0 Window의 크기 조절 가능 여부를 지정한다.
scrollbars yes, no, 1, 0 스크롤바의 보임 여부를 지정한다.
status yes, no, 1, 0 상태 표시줄의 보임 여부를 지정한다.
titlebar yes, no, 1, 0 제목 표시줄의 보임 여부를 지정한다.
toolbar yes, no, 1, 0 도구모음의 보임 여부를 지정한다.
top 픽셀 Window의 위쪽 위치(y 좌표)를 지정한다. IE에서만 사용가능.
width 픽셀 Window의 너비를 지정한다. 최소값은 100이다.
replace 지정된 URL의 히스토리(방문 기록) 정보를 히스토리 리스트에 새로 추가할 것인지
아니면 현재의 히스토리 정보에 덮어 쓸 것인지(replace할 것인지)를 지정한다. 지정 가능한 값은 아래와 같다.
  • true : 지정된 URL의 정보를 현재의 히스토리 정보에 덮어쓴다.
  • false : 지정된 URL의 정보를 히스토리 리스트에 새로 추가한다.
print() Window에 있는 내용을 프린트한다.
prompt() 문자열을 입력 받을 수 있는 입력창을 띄운다.
setInterval() 지정된 코드를 지정된 시간(밀리초) 간격으로 반복 실행되도록 한다.
setTimeout() 지정된 코드를 지정된 시간(밀리초) 이후에 실행되도록 코드의 실행을 연기한다.

Location 객체

Location 객체의 주요 속성 및 메서드
속성 설명
hash URL의 anchor 부분을 반환한다.
예를 들어 URL이 http://localhost:27986/17-20-1.html#part1 이라면
해시 기호(#)를 포함한 #part1을 반환한다.
host URL의 호스트명과 포트 번호를 반환한다.
hostname URL의 호스트명을 반환한다.
href 전체 URL을 반환한다.
pathname URL의 웹 문서 경로를 반환한다.
port 포트 번호를 반환한다.
protocol URL의 프로토콜을 반환한다.
search URL의 쿼리 문자열(query string) 부분을 반환한다.
예를 들어 URL이 http://localhost:27986/17-20-1.html?userID=aaaa 라면
물음표(?)를 포함한 ?userID=aaaa를 반환한다.
메서드 설명
assign() 새로운 웹 문서를 로드한다.
reload() 현재의 웹 문서를 다시 리로드(reload)한다.
reload() 메서드는 기본적으로 웹 브라우저 캐시에 있는 현재의 웹 문서를 다시 로드하는데
reload(true)와 같이 true를 인수로 전달하면 현재의 웹 문서를 웹 서버에서 다시 로드한다.
replace() 현재의 웹 문서를 새로운 웹 문서로 변경한다.
replace() 메서드와 assign() 메서드 모두 새로운 웹 문서를 로드한다는 점에서는 동일한 기능을 하지만
replace() 메서드는 새로운 웹 문서의 정보를 현재의 웹 문서 히스토리(방문 기록) 정보에 덮어쓰므로
다시 이전 웹 문서 즉, 현재의 웹 문서로 history.back()할 수 없다.
반면에 assign() 메서드는 새로운 웹 문서의 정보를 히스토리(방문 기록) 리스트에 새로 추가하므로
다시 이전 웹 문서 즉, 현재의 웹 문서로 history.back()할 수 있다.

Screen 객체

Screen 객체의 주요 속성
속성 설명
availHeight 사용 가능한(available) 화면의 높이를 반환한다. 단, 작업 표시줄은 제외.
availWidth 사용 가능한(available) 화면의 너비를 반환한다. 단, 작업 표시줄은 제외.
colorDepth 사용 가능한 색상의 수를 반환한다.
height 화면의 전체 높이를 반환한다.
pixelDepth 화면의 한 픽셀당 비트 수를 반환한다.
width 화면의 전체 너비를 반환한다.

Navigator 객체

Navigator 객체의 주요 속성
속성 설명
appCodeName 웹 브라우저의 코드 이름을 반환한다.
appName 웹 브라우저의 이름을 반환한다.
appVersion 웹 브라우저의 버전 정보를 반환한다.
cookieEnabled 웹 브라우저의 쿠키 사용 여부를 반환한다.
onLine 웹 브라우저의 인터넷 연결(온라인) 여부를 반환한다.
인터넷에 연결되어 있다면 true를 반환한다.
platform 웹 브라우저가 실행되고 있는 시스템의 환경 정보를 반환한다.
userAgent user-agent 헤더의 값을 반환한다.
